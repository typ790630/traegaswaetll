═══════════════════════════════════════════════════════════

❌ 錯誤信息：
   "AA21 didn't pay prefund"
   "Smart Account does not have sufficient funds"

🎯 根本原因：
   智能賬戶的 BNB 餘額不足，無法支付 prefund（預付費）

✅ 快速解決（3 個步驟）：

┌─────────────────────────────────────────────────────────┐
│ 步驟 1: 獲取智能賬戶地址                                    │
├─────────────────────────────────────────────────────────┤
│ • 方法 A: 在轉賬頁面查看「智能賬戶 BNB 餘額」提示框           │
│ • 方法 B: 運行 check-smart-account.bat                   │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ 步驟 2: 充值 BNB 到智能賬戶                                │
├─────────────────────────────────────────────────────────┤
│ • 使用 MetaMask / Trust Wallet / 幣安交易所               │
│ • 網絡：BSC (BEP20)                                      │
│ • 金額：至少 0.005 BNB（建議 0.01 BNB）                   │
│ • 收款地址：您的智能賬戶地址（從步驟 1 獲取）                │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│ 步驟 3: 等待確認後重試                                      │
├─────────────────────────────────────────────────────────┤
│ • 等待 1-3 分鐘讓交易確認                                  │
│ • 重新打開轉賬頁面                                         │
│ • 檢查「智能賬戶 BNB 餘額」是否 ≥ 0.001 BNB               │
│ • 如果餘額充足，「發送」按鈕會變成可點擊狀態                  │
└─────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════
    💡 重要提示
═══════════════════════════════════════════════════════════

1. EOA 地址 ≠ 智能賬戶地址
   • EOA：您的原始錢包地址
   • 智能賬戶：由 EOA 控制的合約錢包地址
   • 需要向「智能賬戶地址」充值！

2. 為什麼需要 BNB？
   • 即使使用 RADRS 支付 gas，智能賬戶仍需要 BNB
   • 這是 ERC-4337 標準的安全要求
   • BNB 用於支付 prefund，RADRS 用於償還 gas

3. 需要多少 BNB？
   • 最低：0.001 BNB（約 0.6 USD）
   • 建議：0.005 BNB（約 3 USD，可支持多次交易）
   • 長期：0.01-0.02 BNB（可支持數十次交易）

4. 確認正確的網絡
   ✅ BSC (BEP20) - 正確
   ✅ Binance Smart Chain - 正確
   ❌ BEP2 - 錯誤（舊網絡）
   ❌ Ethereum - 錯誤（不同鏈）
